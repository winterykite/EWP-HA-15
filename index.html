Git-Workflow

Mithilfe von Git kann man, anders als zum Beispiel in Google Docs, nicht nur dynamisch, sondern auch offline Projekte bearbeiten, und sie mit der Arbeit von Projektmitgliedern synchronisieren. Dies funktioniert wie folgt:

  • Mithilfe des git clone-commands wird das Projekt auf den Rechner der bearbeitenden Person kopiert, und somit lokal und offline verwendbar. Dieser Klon ist, bis er synchronisiert wird, von dem zentral gespeicherten Projekt unabhängig.
  • Der git add-command merkt vor, welche der Dateien im Workspace geändert werden sollen.
  • Die Änderungen in diesen vorgemerkten Dateien werden mithilfe des commands git commit dann gespeichert. Der git commit-command ist reversibel.
  • Um das lokale und das zentrale Repositorium zu synchronisieren, werden die commands git push und git pull verwendet. git push gibt die lokalen Daten an das zentrale Repositorium weiter, während git pull eventuell veränderte Daten aus dem zentralen Repositorium abruft.
$ git clone https://github.com/winterykite/EWP-HA-15 $ git add index.html $ git commit -m "Index aktualisiert" $ git push [origin] [master] $ git pull
